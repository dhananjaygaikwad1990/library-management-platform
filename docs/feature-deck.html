05 · Circulation workflow

Borrowing and returning update every downstream view

Return copy

Borrow transaction

  • Validates due date
  • Selects first available physical copy
  • Creates the borrow record
  • Sets copy status to borrowed

Return transaction

  • Verifies record ownership
  • Records today's return date
  • Finalizes overdue fine
  • Sets copy status to available

Fine rule

Outstanding fine = overdue days × 1.00. Active overdue fines are calculated dynamically; returned-loan fines stop accruing and can be cleared.

12 · API reference

Members, borrowing, returns & fines

MethodPathPurposeRules
GET/membersList/search membersLibrarian/admin; name or exact ID search
POST/membersCreate memberLibrarian/admin; unique email
POST/borrowBorrow by book or copyMember comes from authenticated email
GET/me/borrowsPersonal loan historyIncludes status, overdue days, fine, title, author
POST/me/borrows/{id}/returnReturn owned loanFinalizes fine and restores availability
POST/me/borrows/{id}/clear-fineClear returned-loan fineLoan must be returned and owned by caller

Ownership protection

The path ID alone is insufficient. The service filters by both borrow_id and the member resolved from the current token, preventing cross-member return or fine operations.

13 · Request & response

Borrow API in detail

Request

POST /borrow
Authorization: Bearer <token>
Content-Type: application/json

{
  "book_id": 2,
  "borrow_date": "2026-07-19",
  "due_date": "2026-08-02",
  "remarks": "Optional note"
}

Successful response

{
  "borrow_id": 41,
  "copy_id": 7,
  "member_id": 1001,
  "borrow_date": "2026-07-19",
  "due_date": "2026-08-02",
  "return_date": null,
  "fine_amount": 0,
  "remarks": "Optional note"
}
  • Use either book_id or copy_id.
  • member_id is deliberately excluded from the request.
  • The service rejects a past due date relative to the borrow date and rejects unavailable inventory.
